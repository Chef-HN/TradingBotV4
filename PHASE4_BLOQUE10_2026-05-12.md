# Fase 4 - Bloque 10 (Paralelo V3->BD->V4 con postgres_tail)

Fecha: 2026-05-12  
Repo: `C:\Users\Abraham\source\repos\TradingBotV4`

## Objetivo del bloque
Permitir paralelo paper V3+V4 usando un solo origen de market data externo: V3 consulta Bybit y V4 consume ticks desde BD (read-only), sin tocar codigo/procesos de V3.

## Implementado
1. Nuevo `MarketDataProvider` en worker Rust:
   - `postgres_tail`
   - Archivo: `worker-rust/src/providers/postgres_tail_market_data.rs`
2. Integracion runtime:
   - `TB_MARKET_DATA_PROVIDER=postgres_tail`
   - aliases: `postgres_ticks`, `v3_ticks`
3. Variables nuevas:
   - `TB_MARKET_DATA_DB_DSN` (default `postgresql://tradingbot:tradingbot@localhost:5433/tradingbotv3`)
   - `TB_MARKET_DATA_DB_TABLE` (default `ticks`)
   - `TB_MARKET_DATA_DB_START_MODE` (`latest`|`oldest`, default `latest`)
4. Seguridad operacional:
   - validacion estricta del identificador de tabla (evita SQL injection en nombre de tabla)
   - conexion lazy/reconnect
   - cursor por `id` para deduplicacion (`id > last_seen_id`)
5. Canary script actualizado:
   - `scripts/phase4_canary_smoke.ps1` ahora soporta `-MarketDataProvider postgres_tail`
   - parametro `-MarketDataDbDsn`
   - `MinCycles` dinamico para `postgres_tail` = `1`
6. README actualizado:
   - documenta `postgres_tail` y comando de corrida paralela.
